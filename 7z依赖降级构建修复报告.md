# 7Z 依赖降级构建修复报告

## 问题

GitHub Actions 构建失败：

```text
commons-lang3-3.14.0.jar: D8: java.lang.NullPointerException
Execution failed for task ':app:mergeExtDexDebug'
```

原因是 `commons-compress:1.26.2` 传递依赖了较新的 `commons-lang3:3.14.0`，当前项目使用的 Android Gradle Plugin 7.2.2 / D8 组合在 dex 该依赖时出现异常。

## 修复

将：

```gradle
implementation 'org.apache.commons:commons-compress:1.26.2'
```

降级为：

```gradle
implementation 'org.apache.commons:commons-compress:1.21'
```

`commons-compress:1.21` 仍包含 `SevenZFile`，可继续用于常见 7Z 解压，同时不会引入 `commons-lang3:3.14.0`。

## 验证

执行：

```bash
./gradlew :app:dependencies --configuration debugRuntimeClasspath
```

依赖树确认变为：

```text
org.apache.commons:commons-compress:1.21
```

不再引入 `commons-lang3`。
