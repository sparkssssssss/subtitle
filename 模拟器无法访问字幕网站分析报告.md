# 模拟器无法访问字幕网站分析报告

## 现象

用户实测：在 Android 模拟器中无法访问字幕网站（项目默认源为 `https://zimuku.org`）。

结合 `adb.txt` 脱敏日志，App 只打印了请求发起日志：

```text
HTTP request url=https://zimuku.org/search?q=...
HTTP request url=https://zimuku.org/search?keyword=...
```

但没有看到：

- `HTTP response code=...`
- OCR 日志
- 验证码提交日志
- 崩溃堆栈

说明请求大概率卡在 DNS / TCP 连接 / TLS 握手 / 等待首包阶段，还没进入验证码或 HTML 解析阶段。

## 代码侧已确认

`app/src/main/AndroidManifest.xml` 已有：

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

并且应用配置了网络安全策略：

```xml
android:networkSecurityConfig="@xml/network_security_config"
```

`network_security_config.xml` 中允许了 `zimuku.org`、`srtku.com` 的明文流量兼容配置。当前主请求使用 HTTPS：

```java
private static final String BASE = "https://zimuku.org";
```

因此目前不像是缺少 Android INTERNET 权限或明文 HTTP 限制导致。

## 可能原因

### 1. 模拟器网络/DNS 无法解析或连接 zimuku.org

如果模拟器浏览器也打不开 zimuku.org，问题不在 App，而在模拟器网络链路。

常见原因：

- 模拟器 DNS 异常；
- host 机器能访问，但模拟器 NAT 网络不能访问；
- VPN/代理只对宿主机浏览器生效，没有透传给模拟器；
- 公司/本地网络策略拦截；
- DNS 污染或解析到不可达 IP。

### 2. 字幕网站对模拟器/机房/异常网络做了限制

zimuku 这类站点经常有防火墙、验证码或访问策略。即使 host 能访问，模拟器出口、User-Agent、TLS 指纹或 IP 段不同，也可能被限流、阻断或长时间不返回。

### 3. HTTPS/TLS 握手问题

如果模拟器系统较旧、证书链异常、代理证书未安装，可能出现 TLS 握手失败。但当前日志没有打印异常类型，需要增加网络异常日志才能确认。

### 4. 日志截取时间偏早

`httpData()` 的读取超时是 30 秒，连接超时是 15 秒。当前日志只看到请求发出，没有最终失败日志；可能日志在完整超时前就停止了。

## 建议排查

### 1. 在模拟器浏览器中分别测试

- `https://zimuku.org`
- `http://zimuku.org`
- `https://www.baidu.com` 或其他确定可访问网站

判断是：

- 只有 zimuku 不通；
- 还是模拟器整体网络不通。

### 2. 用 adb shell 测试 DNS/网络

可执行：

```bash
adb shell getprop net.dns1
adb shell ping -c 3 zimuku.org
adb shell ping -c 3 8.8.8.8
```

如果模拟器有 toybox/wget/curl，也可以：

```bash
adb shell toybox wget -S -O - https://zimuku.org
```

### 3. 如果宿主机用了代理/VPN，要给模拟器配置代理

Android 模拟器通常不会自动继承宿主机浏览器代理。可在模拟器 Wi-Fi 高级设置中配置代理，或启动模拟器时指定代理。

### 4. App 内增加异常日志

建议在 `SearchTask` 的每个搜索 URL catch 里打印异常类型：

```java
Log.d(TAG, "search url failed=" + redactUrl(url)
        + " error=" + e.getClass().getSimpleName() + ": " + e.getMessage());
```

这样下一次 adb 日志能直接看到是：

- `SocketTimeoutException`
- `UnknownHostException`
- `SSLHandshakeException`
- `ConnectException`

## 结论

当前最可能不是 OCR 逻辑问题，也不是 App 权限问题，而是模拟器到 `zimuku.org` 的网络链路问题。需要先确认模拟器浏览器/adb shell 是否能访问该站点，再决定是配置模拟器代理/DNS，还是在 App 中增加备用域名、代理配置或更详细的网络诊断日志。
